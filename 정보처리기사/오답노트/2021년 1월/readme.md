- OSI 7계층
물리 계층 : 전선, 전파, 광섬유, 동축케이블, 모뎀 / 물리적인 송수신 (0과 1)
데이터 링크 계층 : 이더넷, ISDN, ATM / 두 장치간에 데이터 주고 받기
네트워크 계층 : IP, ICMP, IGMP, RARP / 목적지까지 최적의 경로로 전달
전송 계층 : TCP, UDP, RTP / 통신하기 위해 오류 검출, 복구, 흐름제어, 중복검사
세션 계층 : TLS SSH, ISO 8327 / 양 끝단 프로세스 통신 관리
표현 계층 : JPEG, MPEG, XDR, ASN1 / 서로 다른 사용자 시스템 간에 번역 담당
응용 계층 : HTTP, SMTP, FTP / 유저가 이용할 수 있게 서비스를 제공하는 계층

- DB 설계 순서
1. 요구사항 수집 분석
  요구사항에 대한 어떤 데이터들이 필요한지, 어떤 기능들이 필요한지 분석

2. 개념적 설계
  핵심 Entitiy 도출
  E-R 다이어그램 작성

3. 논리적 설계
   목표 DBMS에 맞는 스키마 설계, 트랜잭션 인터페이스를 설계하는 정규화 과정
   
4. 물리적 설계
   특정 DBMS 성능을 고려하여 데이터베이스에 저장할 수 있게 테이블 및 인덱스 설계
   명세서, 테이블 정의서
   
5. 구현 (스킵)

- 요구사항 분류
  1. 기능적 요구사항 : 기능, 서비스
